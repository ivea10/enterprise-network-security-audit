SRIE > TP1 - Reconnaissance / Footprint

The key to successfully exploiting or infiltrating a remote system lies in gathering information. The initial step in the penetration testing is reconnaissance, which involves using tools to retrieve information from the target system.

In this lab, you will learn how to use social engineering techniques to obtain information that can be used to gain access to user passwords and servers. Reconnaissance plays a crucial role in achieving our goals, as it helps us understand the security posture of the target system. The more we learn about the target's security measures, the better we can understand their practices and frequency of security checks.

Reconnaissance is a systematic and methodical process that provides a comprehensive profile of the target's security environment, allowing us to narrow down the potential attack vectors. This includes identifying specific domains, subdomains, or IP ranges to focus our efforts on, thus reducing the overall attack surface.

Useful tools used in this phase

Below is a list of tools that can be used for this pentesting phase:

  1. IP Addresses (private and public)

    Internet Protocol (IP) address refers to a unique numerical label assigned to a networked device, such as a computer, server, router, or other networked device, that uses the IP for communication within an IP-based network. IP addresses are used to uniquely identify and locate devices on a network, allowing them to send and receive data packets.

  2. WHOIS

    whois command is a network utility tool used to query databases to retrieve information about the registration of a domain name, IP address, or autonomous system.

  3. TheHarvester

    TheHarvester is an open-source intelligence gathering tool designed to help security professionals and ethical hackers collect information about a target, such as email addresses, domain names, subdomains, hostnames, IP addresses, and more, from public sources.
